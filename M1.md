# M1 — Data + EDA: Task Breakdown

Derived from PLAN.md §3 (E0) and §13 (tasks 1–3). Scope: **M1 only** — corpus acquisition, cleaning, stratified sampling, EDA, datasheet. No retrieval, no benchmark, no modeling.

GPU note: all tasks below are CPU-only (pandas/matplotlib/language-id). Your 3050 laptop GPU is not a factor for M1 — GPU/VRAM constraints only become relevant from M3 (dense embeddings) onward and should be reassessed then against the plan's 8GB/4060 assumption.

Sequencing: T1.1 → T1.2 → T1.3 → T1.4 → T1.5 → T1.6 → T1.7 → T1.8 are strictly sequential (each depends on the previous). T1.9–T1.11 can run in parallel once T1.8 is done. T1.12 runs last (pulls numbers from EDA + manifest).

---

### T1.1 — Repo scaffold + environment
**Objective:** Create the directory skeleton and reproducible environment so all later data work has somewhere to live.
**Files to create:**
- `environment.yml` (or `pyproject.toml`)
- `configs/data.yaml`
- `src/cragb/__init__.py`, `src/cragb/utils/__init__.py`, `src/cragb/utils/seeds.py`, `src/cragb/utils/io.py`
- `data/raw/.gitkeep`, `data/processed/.gitkeep`, `reports/figures/.gitkeep`
- `.gitignore` (ignore `data/raw`, `data/processed`)
**Python packages:** `pandas`, `pyarrow`, `numpy`, `pyyaml`, `tqdm`, `matplotlib`, `jupyter`
**Expected outputs:** Installable conda/pip env; empty package importable (`import cragb`).
**Verify:** `conda env create -f environment.yml && python -c "import cragb"` runs without error.
**Git commit:** `chore: scaffold repo, env, configs, seed util`

---

### T1.2 — Acquire raw dataset (download + inventory)
**Objective:** Get Amazon Reviews 2023 Clothing/Shoes/Jewelry (reviews + metadata) onto disk and confirm file integrity.
**Files to create:**
- `src/cragb/data/download.py` (script, not full pipeline yet — just fetch + checksum)
- `data/raw/README.md` noting source URLs, download date, file sizes
**Python packages:** `requests` or `huggingface_hub` (whichever the dataset host requires), `hashlib` (stdlib)
**Expected outputs:** `data/raw/reviews_clothing.jsonl(.gz)`, `data/raw/meta_clothing.jsonl(.gz)`
**Verify:** File sizes match dataset card; `gzip -t` (or equivalent) passes; row count via streamed count matches published figure order-of-magnitude.
**Git commit:** `chore(data): download raw reviews + metadata`

---

### T1.3 — Streaming reader + schema probe
**Objective:** Write a memory-safe streamed reader for the raw JSONL files and confirm the schema (fields present, types) before writing any filtering logic.
**Files to create:**
- `src/cragb/data/load.py` (generator-based reader, chunked)
- `notebooks/00_schema_probe.ipynb` (thin — just calls loader, prints first N records, dtypes, null counts)
**Python packages:** `pandas`, `pyarrow`, `ijson` (optional, for true streaming JSON)
**Expected outputs:** Printed schema summary; confirmed field names (`rating`, `text`, `images`, `parent_asin`, `title`, etc.)
**Verify:** Notebook runs top-to-bottom on a 10k-row sample without loading full file into RAM (check memory stays flat via Task Manager).
**Git commit:** `feat(data): streaming loader + schema probe`

---

### T1.4 — Cleaning filters (dedup, language, empty-text, whitespace)
**Objective:** Implement the filter functions from §3 E0 preprocessing as pure, testable functions.
**Files to create:**
- `src/cragb/data/clean.py` (`dedup()`, `filter_language()`, `drop_empty_text()`, `normalize_whitespace()`)
- `tests/test_clean.py` (unit tests with small synthetic DataFrames)
**Python packages:** `pandas`, `langdetect` or `fasttext` (language ID), `pytest`
**Expected outputs:** Passing unit tests; functions importable from `src.cragb.data.clean`.
**Verify:** `pytest tests/test_clean.py -v` all green.
**Git commit:** `feat(data): cleaning filters + unit tests`

---

### T1.5 — Join reviews ↔ metadata on `parent_asin`
**Objective:** Merge review records with product metadata (title, category, attributes), and detect/report join loss (Risk from PLAN.md: "silent join loss on parent_asin").
**Files to create:**
- `src/cragb/data/join.py`
- `tests/test_join.py`
**Python packages:** `pandas`
**Expected outputs:** Joined DataFrame sample; a printed/logged join-loss percentage.
**Verify:** Run on 10k-row sample, confirm join-loss % is logged and reasonable (<5%, or explicitly justified if higher).
**Git commit:** `feat(data): join reviews to metadata on parent_asin`

---

### T1.6 — Per-review feature computation (token length, image parsing)
**Objective:** Compute per-review token length and parse image URL fields, needed for stratification and EDA.
**Files to create:**
- `src/cragb/data/features.py` (`compute_token_length()`, `parse_image_urls()`, `has_image()`)
**Python packages:** `pandas`, `tiktoken` or simple whitespace tokenizer (justify choice)
**Expected outputs:** DataFrame with new columns `token_len`, `n_images`, `has_image`.
**Verify:** Spot-check 5 rows manually against raw text/image field.
**Git commit:** `feat(data): token length + image feature extraction`

---

### T1.7 — Stratified sampling logic
**Objective:** Implement the stratified sampler (by rating bucket, image presence, sub-category) with a fixed seed, addressing Risk C (random sampling starves negatives).
**Files to create:**
- `src/cragb/data/sample.py` (`stratified_sample(df, config, seed)`)
- `tests/test_sample.py` (check output strata proportions vs config targets)
**Python packages:** `pandas`, `numpy`
**Expected outputs:** Function producing a sampled DataFrame matching target strata weights within tolerance.
**Verify:** `pytest tests/test_sample.py` passes; on synthetic data, sampled 1★/2★ share meets configured minimum floor.
**Git commit:** `feat(data): stratified sampling with seed control`

---

### T1.8 — End-to-end corpus build pipeline → `corpus_v1`
**Objective:** Wire T1.3–T1.7 into one script/notebook that produces the frozen `corpus_v1.parquet` with a manifest (row count, hash, schema).
**Files to create:**
- `src/cragb/data/build_corpus.py` (CLI entrypoint, reads `configs/data.yaml`)
- `notebooks/02_corpus_build.ipynb` (thin wrapper, calls `build_corpus.py`)
- `data/processed/corpus_v1.parquet` (output, git-ignored but hash tracked)
- `data/processed/corpus_v1_manifest.json` (row count, sha256 hash, seed, schema, filter counts)
**Python packages:** `pandas`, `pyarrow`, `hashlib` (stdlib)
**Expected outputs:** `corpus_v1.parquet` + manifest committed (manifest only, not the parquet itself if large).
**Verify:** Re-running the script with the same seed reproduces identical row count and hash.
**Git commit:** `feat(data): end-to-end corpus_v1 build + manifest`
